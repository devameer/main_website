## ما هي القيم الافتراضية للـProps في React؟

في React، الـprops هي البيانات الممرّرة من المكوّن الأب إلى المكوّن الابن. وتتيح لك تخصيص كيفية تصرّف المكوّن أو مظهره.

قيمة الـprop الافتراضية هي قيمة بديلة تُستخدم عندما لا تُمرَّر الـprop.

مثلًا، تخيّل أنّ لديك مكوّن `Avatar`. قد ترغب أن يستخدم حجمًا افتراضيًا عندما لا يُمرَّر حجم مخصّص:

```jsx
function Avatar({
  size = 40,
  alt = 'User',
  src
}) {
  return <img src={src} alt={alt} width={size} />;
}
```

في هذا المكوّن،

```jsx
size = 40
```

تعني أنّه إذا لم يمرّر الأب prop باسم `size`، فإنّ المكوّن يستخدم `40`.

و

```jsx
alt = 'User'
```

تعني أنّه إذا لم يمرّر الأب prop باسم `alt`، فإنّ المكوّن يستخدم `"User"` كنص بديل.

أمّا الـprop `src` فليس لها قيمة افتراضية في هذا المثال. وهذا مهم. فليس كل prop يجب أن تكون لها قيمة افتراضية. بعض القيم قد تكون مطلوبة لكي يكون للمكوّن معنى.

هذه هي الفكرة الجوهرية وراء استخدام القيم الافتراضية للـprops في React بشكل صحيح: القيم الافتراضية مفيدة للمدخلات الاختيارية، لا لإخفاء البيانات المطلوبة المفقودة.

## لماذا تُهمّ القيم الافتراضية للـProps

القيم الافتراضية للـprops ليست مجرد مسألة رفاهية. فهي تؤثّر على مدى قابلية تنبّؤ مكوّناتك.

يمكن استخدام مكوّن ذي قيم افتراضية مُصمَّمة جيّدًا بإعداد أقل:

```jsx
<Avatar src="/users/ameer.png" />
```

لا يحتاج الأب إلى تمرير `size` في كل مرّة. ولا يحتاج إلى تمرير `alt` في كل مرّة إذا كانت قيمة بديلة آمنة مقبولة. فالمكوّن يعرف بالفعل كيف يتعامل مع تلك القيم الاختيارية.

هذا يجعل المكوّن أسهل في إعادة الاستخدام.

لكن القيم الافتراضية قد تُسبّب مشاكل أيضًا عند استخدامها بإهمال. مثلًا، إذا اخترع المكوّن بصمت عنوانًا أو معرّف مستخدم أو سعرًا أو قيمة مدفوعة من واجهة برمجية، فقد يخفي مشكلة بيانات حقيقية.

ينبغي أن تجيب القيمة الافتراضية على سؤال بسيط:

هل يمكن لهذا المكوّن أن يتصرّف بشكل صحيح إذا كانت هذه القيمة مفقودة؟

إذا كانت الإجابة نعم، فقد تكون القيمة الافتراضية مفيدة.

إذا كانت الإجابة لا، فمن الأرجح أن تبقى القيمة مطلوبة.

## FAQ

### ما هي القيم الافتراضية للـprops في React؟

القيم الافتراضية للـprops هي قيم بديلة تُستخدم عندما لا يستقبل المكوّن prop معيّنة. وفي المكوّنات الدالّية، تُكتب غالبًا باستخدام قيم JavaScript الافتراضية في توقيع الدالة.

```jsx
function Button({ variant = 'primary' }) {
  return <button>{variant}</button>;
}
```

### هل يجب أن يكون لكل prop في React قيمة افتراضية؟

لا. فقط الـprops الاختيارية يجب أن يكون لها قيم افتراضية. أمّا البيانات المطلوبة فينبغي عادةً أن تبقى مطلوبة حتى لا تُخفى البيانات المفقودة.

### أين يجب أن أحدّد القيم الافتراضية للـprops؟

بالنسبة للمكوّنات الدالّية، فالنمط النظيف هو تحديد القيم الافتراضية في معامل الدالة المُفكَّك:

```jsx
function Card({ title = 'Untitled' }) {
  return <h2>{title}</h2>;
}
```

هذا يبقي القيم الافتراضية مرئية عند حدود المكوّن.

### لماذا يجب أن أتجنّب تغيير الـprops؟

الـprops مدخلات من المكوّن الأب. وتغييرها يجعل المكوّن أصعب في الفهم وقد يُسبّب سلوكًا مربكًا. تعامل مع الـprops كبيانات للقراءة فقط.

### هل `props.title || 'Untitled'` مثل `title = 'Untitled'`؟

لا. عامل `||` يرجع لقيمة بديلة عند أي قيمة falsy مثل السلسلة الفارغة أو `0` أو `false`. أمّا المعامل الافتراضي فيُطبَّق عندما تكون القيمة `undefined`.
